## Pourquoi ce module ?

L'administration manuelle atteint ses limites dès que le parc dépasse quelques dizaines d'appareils. Ce module vous donne les outils pour automatiser l'ensemble du cycle de vie des Mac : de l'enrôlement Zero-Touch à la remédiation automatique des non-conformités, en passant par l'API REST Jamf Pro et le scripting avancé.

Les compétences de ce module correspondent au niveau avancé de la Jamf 200 Course et expert de la Jamf 300 Course. Ce sont les compétences les plus demandées sur le marché pour les ingénieurs Jamf.

## Ce que vous allez apprendre

À la fin de ce module, vous serez capable de :

- **Exploiter** l'API REST Jamf Pro v2 pour automatiser n'importe quelle opération
- **Écrire** des scripts Bash robustes adaptés à l'environnement Jamf
- **Créer** et distribuer des packages PKG de qualité production
- **Concevoir** un workflow Zero-Touch complet avec des PreStage Enrollment
- **Implémenter** la remédiation automatique des politiques non conformes
- **Gérer** les mises à jour macOS de façon sécurisée et contrôlée

## L'API REST Jamf Pro v2

Jamf Pro expose une API REST complète documentée via Swagger. Elle permet de :

```bash
# Obtenir un token d'authentification Bearer
TOKEN=$(curl -s -X POST \
  "https://jamf.example.com/api/v1/auth/token" \
  -H "Authorization: Basic $(echo -n 'user:password' | base64)" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['token'])")

# Lister tous les Mac
curl -s -H "Authorization: Bearer $TOKEN" \
  "https://jamf.example.com/api/v1/computers-preview" \
  | python3 -m json.tool
```

L'API v2 supporte aussi les webhooks pour les intégrations temps réel avec Slack, JIRA ou ServiceNow.

## Scripting Bash pour Jamf — Bonnes pratiques

```bash
#!/bin/bash
# Template de script Jamf Pro production-ready

# Paramètres Jamf (positions 4, 5, 6)
JAMF_PARAM4="$4"  # URL du serveur applicatif
JAMF_PARAM5="$5"  # Mode debug
JAMF_PARAM6="$6"  # Option supplémentaire

# Logging centralisé
LOG_FILE="/var/log/jamf-custom.log"
log() { echo "$(date '+%Y-%m-%d %H:%M:%S') - $1" | tee -a "$LOG_FILE"; }

# Gestion d'erreurs
set -e
trap 'log "ERREUR ligne $LINENO — exit $?"' ERR

log "Début du script"
# ... votre logique ici
log "Script terminé avec succès"
exit 0
```
